# Contributing Guide

Thank you for your interest in contributing to this project! We believe this is a great community-driven project, and everyone is welcome to contribute according to their individual needs and expertise.

## Getting Started

We appreciate your contributions and want to make the process as smooth as possible. Here are some guidelines to help you get started:

## Technical Guidelines

- **Type Conversion**: Currently, this project doesn't use `trait` to constrain type conversion. Please use `fnalias @vscode.coerce` to import type coercion functions.

- **Pull Request Requirements**: When creating a PR, please include the concrete source location for the specification in [vscode.d.ts](./vscode.d.ts) or [vscode-no-comment.d.ts](./vscode-no-comment.d.ts) for review purposes.

- **Missing Methods**: Some types might be missing methods or static members. Please feel free to add them to the related files.

## Project Status

This project is currently in an early stage of development. Please don't worry about API breaking changes - we will work together to fix them quickly and publish new versions as needed.

## How to Contribute

We welcome all forms of contribution, including but not limited to:
- Bug fixes
- Feature additions
- Documentation improvements
- Code reviews
- Testing

Thank you for helping make this project better for everyone!
